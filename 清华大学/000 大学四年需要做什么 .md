### 面向“uCore”的筑基期学习路线图

你可以按以下四个阶段推进，尤其侧重前两个阶段。

-   **阶段一：C语言功力的极致打磨（耗时：2-4周）**
    -   **目标**：能非常熟练地使用指针、结构体、位运算、函数指针等。
    -   **行动**：结合郑莉老师的课，重点学习指针和内存操作部分。可以找一个用C实现简单程序的代码，比如“学生管理系统”，亲手写一遍。

-   **阶段二：计算机原理与数据结构并进（耗时：4-6周）**
    -   **目标**：1. 彻底搞懂“存储体系”和“中断”；2. 能手写出链表、栈、队列等基础结构。
    -   **行动**：看刘卫东老师课程中的存储、中断部分。同步学习邓俊辉老师《数据结构》的向量、列表部分。

-   **阶段三：汇编与链接的补充（穿插学习）**
    -   **目标**：能读懂简单的x86汇编，理解程序如何被编译链接。
    -   **行动**：不必找专门课程，在遇到不懂的汇编指令时，直接搜索即可。

-   **阶段四：向操作系统发起总攻**
    -   **行动**：完成以上准备后，正式进入你找到的《操作系统》课程和 **uCore 实验**。

这条路很漫长，但每一步都扎实。不必追求速度，深刻理解远比“学完了”重要。清华的学生也是用了整整四年去吸收这些内容的。当你坚持不下去时，就想想那些让你敬佩的清华毕业生，他们也是这样一步步走过来的。

### 教材

* 计算机原理：搜索 “计算机组成原理 刘卫东” 或 “计算机组成 清华大学”

* C语言：搜索 “C++语言程序设计 郑莉” （清华的C语言课程常以C++为名，内容覆盖C语言基础）

* 数据结构：搜索 “数据结构 邓俊辉” （这门课分上下两册，都很经典）

* ### 注意事项

如果你的首要目标是**顺利攻克 uCore 操作系统实验**，那答案是：**先学 C 语言，而且必须学到能熟练操作指针和内存的程度。**

但这不意味着你要先把 C 语言完全学精再去碰别的。最有效的是“**C 语言为主，计算机原理为辅**”的穿插式学习。下面是具体原因和操作方式。

#### 为什么是这个顺序？

**1. C 语言是“战场的武器”**
uCore 实验的每一步，都是用 C 和少量汇编在写代码。如果对指针、内存操作不熟，你在实验里会寸步难行。这个门槛绕不过去。

**2. 原理结合实践，理解才深刻**
反过来，如果只学 C 语言，你可能不理解“为什么指针指向的这个地址就是硬件寄存器？”这类问题。
这时，就需要去刘卫东老师的《计算机组成原理》里找答案，比如看“内存映射 I/O”这一节。这种**带着实际问题去学原理**的方式，效率最高。

#### 具体执行：一个“三明治”学习法

你可以这样安排，让两者相辅相成：

-   **第一层（先啃 C 语言基础）**
    -   **目标**：把 C 语言学到**指针和结构体**的章节，搞懂指针的算术运算、指针与数组的关系、结构体内存布局。
    -   **资源**：搜 **“C语言程序设计 指针”** 等关键词，找专项练习。比如可以试试用 C 语言写一个简单的“学生信息管理系统”，用链表来实现。

-   **第二层（再切入原理重点）**
    -   **目标**：当你在 C 语言里操作 `malloc` 和指针时，会好奇系统如何管理内存，这时切入原理。
    -   **资源**：看刘卫东课程中，**存储系统**（内存、Cache）、**中断与异常**、**总线** 这几个部分。不用看 CPU 流水线设计等过于细节的内容。

-   **第三层（用 uCore 实验来检验）**
    -   **行动**：当你 C 语言的基础语法熟练，且理解了内存和中断的原理后，就可以直接上手 uCore 的 `lab1`。实验中需要更多数据结构（如链表、哈希表）时，再去学邓俊辉老师的《数据结构》对应章节。

**总结一下**：
你的学习路径可以是 **C语言基础 → 原理课中的存储/中断专题 → uCore实验（并同步按需学习数据结构）**。

现在可以先去搜 “C语言 指针详解” 或直接开始看郑莉老师的 C++ 课（重点看 C 语言共性部分），马上开始写代码。当你在内存操作上感到困惑时，再去打开刘卫东老师的课程专题学习，那时你会豁然开朗。

**现在的《C++语言程序设计（第5版）》和配套课程，确实已经完全以 C++ 标准语法和面向对象为主线了。**

不过，这依然是为你打下坚实 C 语言基础的最好教材。**用“面向对象”的教材学“面向过程”的底层功夫，反而能让你对底层的理解更深刻。** 关键就在于，你要主动聚焦和取舍。

#### 为什么它仍是 uCore 实验的最好“马步桩”？

-   **C++ 是 C 的超集**：这门课的前半部分，本质上就是在用更严格的编译器教 C 语言。它覆盖了所有你需要的基础。
-   **被迫理解得更透彻**：C++ 严格的类型检查会逼着你写代码更规范。比如，C 语言里 `malloc` 返回的指针可以隐式转换，C++ 则必须强制转换。这种“麻烦”会让你对指针类型理解得更精准，这对 uCore 那种直接操作物理地址的代码是极大的保护。
-   **“封装”思维是共通的**：uCore 内核里使用结构体和函数指针来模拟“类”和“接口”（如文件系统、进程调度）。学懂 C++ 的封装，你回头再看 uCore 那些层层抽象的设计，会有恍然大悟的感觉。

请把这本书拆开，明确哪些是当前刚需，哪些是未来储备。

| 章节内容 | 与你目标的关系 | 学习策略 |
| :--- | :--- | :--- |
| **第1-3章：基础与函数** | **【uCore 核心刚需】** <br>数据类型、流程控制、函数是任何编程的基础。 | **重点掌握**。尤其要搞懂函数调用栈和参数的传递机制（值传递、指针传递）。 |

| **第6章：结构体与联合体** | **【uCore 核心刚需】** <br>内核里所有的核心对象（进程、页面、文件）都是一个结构体。 | **必须精通**。重点理解结构体在内存中的布局和字节对齐，这对理解硬件交互至关重要。 |

**给你的执行建议**：学习时，C++ 独有的对象输入输出（`cout`）可以先放，沿用 C 语言的 `printf`，它能让你对格式化输出（八进制、十六进制、地址）掌握得更好。等到写应用时，再转用更方便的 `cout`。

这本教材完全担得起你“筑基”的任务。就按这个顺序，先把前6章吃透，尤其是数组、指针和结构体。当你把基础打扎实了，开始看计算机组成原理或者尝试 uCore 实验时，可以随时回来问我。
